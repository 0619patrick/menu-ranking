## 一、产品是什么

一款**给餐饮集团运营老板用的销量数据自动化整理工具**。

把各家门店从 POS 系统导出的 Excel 文件上传进来，工具会按照标准菜单自动归类、汇总、排序，并生成可下载的「菜品销量对照表」Excel。

### 1.1 解决的核心问题

**月度汇总销量是餐饮老板的高频痛点**：
- 每家店每月导出 POS Excel（4 种不同格式）
- 人工对照菜单标准分类、整理销量排行
- 多店横向对比需要手动拼贴
- 单店 1-2 小时、多店多月份起步半天

**本工具一键解决**：上传 → 自动归类 → 多店并排 → Excel 下载。**整套流程 < 5 分钟**。

### 2.2 核心功能列表

#### 🍱 数据处理
- 上传 4 种 POS 格式 Excel（实装 2 种：餐飲王 + 美團）
- 美團数据自动按 (大类, 菜名, 规格) 聚合
- 繁/简体列名自动识别
- 退菜数据自动忽略（金额已归零）
- 同名菜在不同 POS 写法自动合并

#### 📋 智能归类
- 菜单标准分类自动匹配
- POS 错放分类强制纠正（FORCE_CAT 规则）
- 套餐分类整体归"堂食菜單外"
- 套餐内的单品饭/面优先归"主食系列"
- POS 原生分类（如阿城内地卤味三档）整段直接以 POS 为准

#### 🏷️ 特殊业务
- **茶位高亮**（黄色背景行首）
- **外卖按平台分组**（Keeta / Foodpanda / Take 等多种标识）
- **補差價单列**（如泰菜的"手寫單"，金额负数）
- **加料拆分显示**（如四季芬芳的米線加料：「點餐 13 / ¥78 + 套餐內含 49 次」）
- **🆕 新菜标注**（菜单 PDF 没列但 POS 有售）

#### 📊 多店对比
- 同 sheet 内多店左右并排（每店 5 列：分類/排序/品名/數量/金額）
- 分类按菜单声明顺序对齐（同分类占同一行段，便于横向比较）
- 类内菜品按金额降序

#### 📂 分组导出
- **按 (地区, 品牌, 月份) 分 Sheet**：如「香港天天3月」「内地天天3月」
- **按品牌分 Excel 文件**：天天 + 阿城一起上传 → 输出 ZIP 含 2 个 .xlsx
- **单品牌直接下载** .xlsx，不打 zip
- Sheet 顺序稳定：品牌按上传顺序，内地 → 香港，月份升序

#### 🌐 用户界面
- 顶部 5 个餐厅 Tab，切换显示对应快速选店按钮
- 每店卡片：店名、餐廳预设、地区、月份、文件
- 餐廳下拉提供 7 种预设组合（覆盖所有现有店）
- 文件支持拖放或点击选择
- 网页预览 + 可校对疑似异常项（▶ 展开 / ✕ 删除）

## 三、典型使用流程

### 3.1 单店生成排行表

1. 浏览器打开 https://menu-ranking.onrender.com
2. 顶部选餐厅 Tab（如「天天」）
3. 点快速选店按钮（如「沙田（餐飲王）」）—— 自动填好店名 + 餐廳预设
4. 选月份（默认当月）
5. 点击「POS 數據文件」区域 → 选择 POS 导出的 xlsx
6. 点「生成預覽」—— 几秒后预览出现在下方
7. 看预览，对照觉得没问题
8. 点「下載 Excel」—— 直接下载 .xlsx

### 3.2 多店混合上传

1. 顶部点对应餐厅 Tab，点快选按钮加店 1
2. 切到另一品牌 Tab，再点快选 → 加店 2
3. 重复加到所有要处理的店
4. 各店各自选月份/地区
5. 点「生成預覽」→ 下方 Tab 切换看每店预览
6. 点「下載 Excel」：
   - 若多品牌 → 下载 ZIP（每品牌一个 .xlsx）
   - 若单品牌 → 直接单 .xlsx
7. ZIP 解压后每个 .xlsx 内含按 (地区+月份) 分的 sheet

### 3.3 手动加店（自定义）

1. 点「+ 再加一家店」—— 默认套当前 Tab 的餐厅预设
2. 「店舖名稱」手动输入（如「香港天天太古城」）
3. 「餐廳」下拉：若快选不对，直接改（7 种预设全在）
4. 地区/月份按需选
5. 上传文件、生成預覽

## 四、特色业务规则详解

### 4.1 多店行级对齐

举例：同时上传天天 沙田 + 太古城，会输出一个 sheet「香港天天3月」，左右两栏：

```
分類       │ 排序 │ 品名         │ 數量 │ 金額    ║ 分類       │ 排序 │ 品名         │ 數量 │ 金額
天天海南雞 │ 1   │ 海南雞(中份) │ 753  │ 49,294 ║ 天天海南雞 │ 1   │ 海南雞拼烧雞 │ 238  │ 69,069
            2   │ 海南雞拼烧雞 │ 319  │ 41,953 ║              2   │ 海南雞(中份) │ 330  │ 52,390
            3   │ 海南雞(大份) │ 89   │ 23,602 ║              3   │ 海南雞(大份) │ 108  │ 30,923
```

两边的"天天海南雞系列"分类**占同一行段**，方便老板对比两家店的销量。

### 4.2 加料「點餐 vs 套餐內含」

四季芬芳的"自選靚湯米線"套餐含一个加料。POS 里同一道加料有 2 行：
- 价 6 元 / 数量 13 = ¥78（顾客单点）
- 价 0 元 / 数量 49 = ¥0（套餐内含送的）

工具自动拆分显示：

```
韭菜 (+$6)  〔套餐內含 49次〕   數量: 13  金額: ¥78
```

收费金额和内含次数都看得到，方便老板看一道加料"卖了多少 + 送了多少"。

### 4.3 POS 原生分类（深圳阿城卤味）

深圳阿城的「12元精选卤味/特色卤味/麻辣卤味/手作系列」在菜单 PDF 上没列具体菜品，但 POS 里有十几个项目。

工具把这 4 个分类设为「POS 原生分类」：
- 不打 🆕 标签（这本来就是该分类的内容，不是"新菜"）
- 全部 POS 项目直接按金额排进段内

### 4.4 套餐路由

POS「单人套餐」「双人套餐」分类下挂着各种"鹅肉饭+卤肉饭"等套餐组合。

- 整段归到"堂食菜單外"（不污染正餐排行）
- 但里头如果有单品饭/面（如「冬陰功汤面」），仍自动归到"主食系列"

## 五、技术架构（简述）

详见 GitHub 仓库 README 和源码注释。这里只列关键结构：

```
┌───────────────────────────────────────────┐
│  浏览器（顶部 tab + 多店卡片 + 预览表）   │
└──────────────────┬────────────────────────┘
                   │ HTTP (POST /preview, /generate)
                   ▼
┌───────────────────────────────────────────┐
│  Flask (server.py) - 路由 + 参数校验      │
└──────────────────┬────────────────────────┘
                   │
       ┌───────────┼───────────┐
       ▼           ▼           ▼
  ┌────────┐  ┌────────┐  ┌──────────┐
  │  POS   │  │ Menu   │  │Transformer│
  │ Adapter│  │ Config │  │ (核心引擎)│
  └────────┘  └────────┘  └──────────┘
   餐飲王     天天/阿城    路由 + 聚合
   美團       泰菜/四季    多店并排
                          Excel 生成
```

**3 层分离**：
1. **POS Adapter**：把任何 POS 格式归一成 4 列标准 DataFrame（项目名称/分类/数量/金额）
2. **Menu Config**：每个餐厅一份 `menus/<name>.py`，定义菜单结构 + 路由规则
3. **Transformer**：通用引擎，不关心 POS 是谁、菜单是哪份，统一处理

**好处**：
- 加新 POS = 写一个 Adapter（不动任何餐厅）
- 加新餐厅 = 写一份 menu config（不动任何 POS）
- 业务规则集中在 menu 文件，开发者一处即可掌握

## 八、已知限制

1. **冷启动延迟**：Render Free 闲置 15 分钟后首次访问要等 30-60 秒
2. **大批量上传可能 OOM**：6-8 家店是 Free Plan 极限，10+ 建议升 Starter
3. **数据不持久化**：每次重新上传文件，不存历史月份
4. **单用户**：无登录、无权限隔离（适合小团队内部使用）
5. **POS 导出仍需手动**：当前不能从 POS API 自动拉数据
6. **菜单更新需手动维护**：新菜上市或改名需修改 `menus/<name>.py`
7. **零售周边 POS 项**：如「外賣盒」「打包盒」等已自动丢弃，但若 POS 新增其他周边项目需在 `drop_names` 中维护

## 九、运维 / 维护指南

### 9.1 加新店

编辑 `app/templates/index.html` 中 `QS_STORES_BY_RESTAURANT`：

```javascript
'tiantian': {
  '香港': [
    { label: '新店名', name: '香港天天新店', pos: 'canyinwang' },
    ...
  ]
}
```

提交推 main，Render 自动部署。

### 9.2 加新菜（菜单更新）

编辑对应餐厅的 `app/menus/<brand>.py`，在 `MENU` 列表加项：

```python
('前菜/小食', [
    ('新菜名', 49, '份', ['POS菜名变体1', 'POS菜名变体2']),
    ...
])
```

### 9.3 加新餐厅

1. 写 `app/menus/<new_brand>.py`（参考已有的 6 份）
2. 在 `app/menus/__init__.py` 注册
3. 在 `index.html` 加顶部 Tab + 快速选店配置

### 9.4 加新 POS 系统

1. 写 `app/pos_adapters/<pos_name>.py`，继承 `PosAdapter`，实现 `load()` 返回标准 4 列 DataFrame
2. 在 `app/pos_adapters/__init__.py` 注册
3. 在 `index.html` 的 `RESTAURANT_PRESETS` 加新组合
